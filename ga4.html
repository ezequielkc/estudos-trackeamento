O que é Google Analytics 4?

Google Analytics 4 (GA4) é a versão mais recente da plataforma de análise do Google, lançada em outubro de 2020. É uma solução baseada em eventos que oferece uma visão unificada do comportamento do usuário em diferentes plataformas e dispositivos.

Conceito Fundamental

O GA4 representa uma mudança de paradigma: de um modelo baseado em sessões (Universal Analytics) para um modelo baseado em eventos, permitindo rastreamento mais flexível e preciso do comportamento do usuário.

Principais Características do GA4:

  • Modelo Baseado em Eventos: Todos os dados são coletados como eventos, oferecendo maior flexibilidade
  • Rastreamento Cross-Platform: Integra dados de web, apps iOS e Android em uma única propriedade
  • Machine Learning Integrado: Insights automáticos e previsões usando IA do Google
  • Privacidade Aprimorada: Conformidade com LGPD e outras regulamentações de privacidade
  • Análise Avançada: Relatórios exploratórios e análises personalizadas
  • Integração com BigQuery: Exportação gratuita de dados para análise avançada

Por que Migrar para GA4?

Futuro da Análise

  • Universal Analytics foi descontinuado
  • GA4 é a plataforma do futuro

Melhor Rastreamento

Privacidade

GA4 vs Universal Analytics

Entender as diferenças entre GA4 e Universal Analytics (UA) é essencial para uma migração bem-sucedida e para aproveitar ao máximo as novas funcionalidades.

Principais Diferenças

O GA4 representa uma mudança fundamental na forma como os dados são coletados, processados e apresentados, exigindo uma nova mentalidade analítica.

Comparação Detalhada:

Característica Universal Analytics Google Analytics 4
Modelo de Dados Baseado em sessões e pageviews Baseado em eventos
Limite de Eventos 500 hits por sessão Sem limite prático
Propriedades Separadas para web e app Unificadas (web + app)
Parâmetros Customizados Limitado (20 custom dimensions) Até 50 parâmetros por evento
Retenção de Dados 14 meses (gratuito) 14 meses (gratuito) ou ilimitado (GA360)
BigQuery Apenas GA360 (pago) Gratuito para todos
Machine Learning Limitado Integrado e avançado

Mudanças Conceituais Importantes:

De Sessões para Eventos

No UA, tudo girava em torno de sessões. No GA4, cada interação é um evento independente, permitindo análises mais granulares e flexíveis.

De Pageviews para Eventos de Página

No GA4, até mesmo visualizações de página são eventos (page_view), dando mais controle sobre o que é rastreado.

De Goals para Conversões

Goals do UA foram substituídos por eventos marcados como conversões no GA4, com muito mais flexibilidade e sem limite de conversões.

Como Configurar o GA4

Configurar o GA4 corretamente é fundamental para obter dados precisos e úteis. Siga este passo a passo completo.

Passo 1: Criar uma Propriedade GA4

  1. Acesse analytics.google.com
  2. Clique em "Administrador" (ícone de engrenagem)
  3. Na coluna "Propriedade", clique em "Criar propriedade"
  4. Preencha o nome da propriedade (ex: "Meu Site - GA4")
  5. Selecione o fuso horário e a moeda
  6. Clique em "Próximo" e configure informações do negócio
  7. Escolha os objetivos do negócio
  8. Clique em "Criar"

Passo 2: Obter o ID de Medição (Measurement ID)

Onde encontrar o Measurement ID:

1. No GA4, vá em Administrador
2. Clique em "Fluxos de dados"
3. Selecione seu fluxo de dados web
4. O Measurement ID estará no topo (formato: G-XXXXXXXXXX)

Passo 3: Instalar o GA4 via Google Tag Manager

Configuração no GTM:

1. No GTM, crie uma nova Tag
2. Tipo: Google Analytics: GA4 Configuration
3. Measurement ID: Cole seu G-XXXXXXXXXX
4. Acionador: All Pages
5. Salve e publique

Passo 4: Configurações Importantes

Configurações de Dados

  • Ativar coleta de dados aprimorada
  • Configurar retenção de dados
  • Definir fuso horário correto
  • Configurar moeda

Privacidade

  • Configurar consentimento de cookies
  • Ativar modelagem de dados
  • Configurar exclusão de IPs
  • Revisar políticas de privacidade

Integrações

  • Conectar Google Ads
  • Conectar Search Console
  • Configurar BigQuery (se necessário)
  • Integrar com outras plataformas

Passo 5: Validar a Instalação

Como Validar:

  1. Use o DebugView no GA4 (Configurações > DebugView)
  2. Ative o modo de depuração no GTM Preview
  3. Navegue pelo site e verifique eventos em tempo real
  4. Confirme que os eventos estão sendo recebidos corretamente
  5. Verifique o relatório "Tempo Real" no GA4

Eventos e Parâmetros no GA4

No GA4, tudo é um evento. Entender como funcionam eventos e parâmetros é essencial para aproveitar ao máximo a plataforma.

Conceito de Eventos

Eventos são ações que os usuários realizam em seu site ou app. Cada evento pode ter até 50 parâmetros customizados para fornecer contexto adicional.

Eventos Automáticos do GA4:

Eventos Coletados Automaticamente:

• first_visit - Primeira visita do usuário
• page_view - Visualização de página
• session_start - Início de sessão
• scroll - Scroll na página (quando ativado)
• click - Cliques em links externos (quando configurado)
• view_search_results - Visualização de resultados de busca
• file_download - Download de arquivos (quando configurado)

Eventos Aprimorados:

Eventos aprimorados são eventos automáticos que podem ser ativados nas configurações do GA4:

  • scroll: Quando usuário rola 90% da página
  • outbound_click: Cliques em links externos
  • site_search: Buscas no site
  • video_start, video_progress, video_complete: Interações com vídeos
  • file_download: Downloads de arquivos

Eventos Personalizados:

Exemplo: Criar Evento Personalizado via GTM

1. No GTM, crie uma Tag
2. Tipo: Google Analytics: GA4 Event
3. Configuration Tag: Selecione sua tag GA4 Configuration
4. Event Name: button_click (ou outro nome)
5. Parâmetros:
   - button_id: {{Click Element}}.id
   - button_text: {{Click Text}}
   - page_location: {{Page URL}}
6. Acionador: Clique em elementos com classe "btn"
7. Salve e publique

Estrutura de Parâmetros:

Exemplo de Evento com Parâmetros via DataLayer:

dataLayer.push({
    'event': 'purchase',
    'transaction_id': 'T12345',
    'value': 99.90,
    'currency': 'BRL',
    'items': [{
        'item_id': 'SKU123',
        'item_name': 'Produto Exemplo',
        'category': 'Eletrônicos',
        'quantity': 1,
        'price': 99.90
    }]
});

Parâmetros Recomendados:

Parâmetros Padrão

  • page_location
  • page_title
  • page_referrer
  • engagement_time_msec

Parâmetros Customizados

  • Até 50 por evento
  • Nomes em snake_case
  • Valores string ou numéricos
  • Registrados como dimensões

Boas Práticas

  • Use nomenclatura consistente
  • Documente todos os parâmetros
  • Evite duplicação de dados
  • Teste antes de publicar

Relatórios e Análises no GA4

O GA4 oferece diferentes tipos de relatórios para analisar o comportamento dos usuários e o desempenho do seu negócio.

Tipos de Relatórios:

Relatórios em Tempo Real

Visualize eventos que estão acontecendo agora mesmo no seu site. Útil para validar implementações e monitorar campanhas em tempo real.

  • Eventos dos últimos 30 minutos
  • Usuários ativos no momento
  • Páginas mais visualizadas
  • Origem do tráfego

Relatórios de Aquisição

Entenda como os usuários chegam ao seu site. Analise canais, campanhas, fontes e mídias.

  • Canais de aquisição
  • Campanhas e UTM parameters
  • Fontes e mídias
  • Comparação de canais

Relatórios de Engajamento

Analise como os usuários interagem com seu conteúdo. Páginas, eventos, conversões e tempo de engajamento.

  • Páginas e telas
  • Eventos principais
  • Conversões
  • Tempo de engajamento

Relatórios de Monetização

Para e-commerce, analise receita, produtos vendidos, transações e funil de compra.

  • Receita e transações
  • Produtos mais vendidos
  • Funil de compra
  • Desempenho por produto

Análises Exploratórias:

Análises Personalizadas

As Análises Exploratórias permitem criar relatórios personalizados usando diferentes técnicas analíticas como análise de caminho, análise de coorte, análise de segmentação e muito mais.

Como Criar uma Análise Exploratória:

  1. No GA4, vá em "Explorar" no menu lateral
  2. Clique em "Criar nova exploração"
  3. Escolha um modelo (Análise de caminho, Análise de coorte, etc.)
  4. Configure dimensões e métricas
  5. Adicione segmentos se necessário
  6. Salve a exploração para uso futuro

Conversões no GA4

No GA4, qualquer evento pode ser marcado como conversão, oferecendo muito mais flexibilidade que o Universal Analytics.

Conversões Flexíveis

Diferente do UA que tinha limite de 20 goals, o GA4 permite marcar quantos eventos quiser como conversões, sem limite.

Como Marcar um Evento como Conversão:

  1. No GA4, vá em "Administrador"
  2. Na coluna "Propriedade", clique em "Eventos"
  3. Encontre o evento que deseja marcar como conversão
  4. Ative o toggle "Marcar como conversão"
  5. O evento agora aparecerá nos relatórios de conversão

Tipos de Conversões Comuns:

Micro-Conversões

  • Cadastro em newsletter
  • Download de material
  • Visualização de vídeo
  • Scroll de 75% da página
  • Tempo na página > 2min

Macro-Conversões

  • Compra realizada
  • Formulário enviado
  • Assinatura ativada
  • Agendamento confirmado
  • Lead qualificado

Configurar Conversões via GTM:

Exemplo: Marcar Envio de Formulário como Conversão

1. No GTM, crie um evento personalizado
2. Event Name: form_submit
3. Configure o acionador para formulários
4. Publique no GTM
5. No GA4, marque o evento como conversão
6. Aguarde até 24h para aparecer nos relatórios

Atribuição de Conversões:

Modelos de Atribuição

O GA4 oferece diferentes modelos de atribuição (Primeiro clique, Último clique, Baseado em dados, etc.) para entender qual canal contribuiu para a conversão.

Janela de Lookback

Configure a janela de lookback (30, 60 ou 90 dias) para determinar quanto tempo antes da conversão um clique ou visualização pode ser atribuído.

Integração GA4 com Google Tag Manager

Integrar GA4 com GTM é a melhor prática para gerenciar tags e eventos de forma centralizada e sem necessidade de alterar código.

Configuração Básica:

Tag GA4 Configuration no GTM:

Tipo: Google Analytics: GA4 Configuration
Measurement ID: G-XXXXXXXXXX
Acionador: All Pages

Configurações Avançadas:
- Campos a definir:
  * page_title: {{Page Title}}
  * page_location: {{Page URL}}
  * page_referrer: {{Page Referrer}}

Enviar Eventos via DataLayer:

Estrutura Recomendada:

// Evento simples
dataLayer.push({
    'event': 'button_click',
    'button_id': 'cta-primary',
    'button_text': 'Comprar Agora'
});

// Evento de conversão
dataLayer.push({
    'event': 'purchase',
    'ecommerce': {
        'transaction_id': 'T12345',
        'value': 199.90,
        'currency': 'BRL',
        'items': [{
            'item_id': 'SKU123',
            'item_name': 'Produto',
            'price': 199.90,
            'quantity': 1
        }]
    }
});

Boas Práticas de Integração:

Nomenclatura Consistente

Use padrões claros para nomes de eventos: action_object (ex: click_button_cta, submit_form_contact)

Validação

Sempre use o Preview Mode do GTM e DebugView do GA4 para validar eventos antes de publicar em produção.

Documentação

Documente todos os eventos, parâmetros e conversões implementados para facilitar manutenção futura.
